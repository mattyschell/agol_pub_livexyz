## From the LiveXYZ team

Your accounts in the [Live XYZ Directory](https://directory.livexyz.com/places) are both active. We use Microsoft SSO and have included [Login Instructions](https://drive.google.com/file/d/1nWXywB92QobddE_AEqz0in6iRrjM__uU/view) for your reference.


To quickly recap some key items discussed:

* API: Live XYZ has a GraphQL API. Here is a link to the [API Instructions](https://docs.google.com/document/d/1-uPcYg8qLm7BbFssycflbYAHdTIyQGt-889zUl0Vh9c/edit?tab=t.0). You can access your API Key from the top right menu of the Live XYZ Directory after logging in

* Data Size & Structure: ~143,000 today records; ~405,000 historical – and growing. 56 current data fields as described in the Live XYZ [Data Dictionary](https://docs.google.com/spreadsheets/d/1pbWq0HXiY8yMPUuKjsDNA_fcJoTI8zJwUHudmPhuWhg/edit?gid=858089682#gid=858089682)


* Unique ID: Each row of Live XYZ data has a unique stateID which can be used for assigning the ArcGIS ID. The stateID represents status of a space (spaceID) over a validity time window (validityTime.start and validityTime.end). Over time, spaceID's accrue multiple stateIDs as they turn over from one occupancy status to the next 


* Update Frequency: Data is updated on a nightly basis in the Directory and API reflecting all data updates captured that day; Live XYZ’s ground survey map team makes thousands of updates a day


* Data Access: All NYC Gov staff can access the data via the OTI contract. Raw data cannot be shared outside of the City. Download access should be strictly limited to NYC Gov staff
